## 2. Agent 交互流程

### 2.1 OA Agent 对话流程
```
用户输入
  │
  ▼
┌──────────────┐    ┌──────────────┐
│ LLM分析意图  │───▶│ 需要工具调用? │
└──────────────┘    └──────┬───────┘
                    是 │       │ 否
              ┌────────▼──┐    ▼
              │ 执行工具   │  直接回复
              │ Function  │
              │ Calling   │
              └────┬──────┘
                   ▼
              ┌──────────────┐
              │ LLM整合结果  │──▶ 返回用户
              └──────────────┘
```

### 2.2 审批Agent工作流（LangGraph）
```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│ 分析请求  │───▶│ 政策审核  │───▶│ 做出决定  │───▶│ 执行决定  │──▶ END
│ (analyze) │    │ (check)  │    │ (decide) │    │ (execute)│
└──────────┘    └──────────┘    └──────────┘    └──────────┘
     │               │               │               │
     ▼               ▼               ▼               ▼
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│评估风险  │    │合规检查  │    │综合决策  │    │更新数据库│
│生成建议  │    │金额校验  │    │输出理由  │    │记录结果  │
└──────────┘    └──────────┘    └──────────┘    └──────────┘
```

## 3. 数据流设计

### 3.1 知识库初始化
```
Markdown文档 → Text Splitter → Embedding → ChromaDB存储
```

### 3.2 RAG问答
```
用户问题 → Embedding → 向量检索 → 拼接上下文 → LLM生成答案 → 返回
```

### 3.3 异常分析
```
报销数据 → 统计分析 → LLM分析 → 异常标记 → 数据库更新 → 报表
```

## 4. Agent设计模式

本项目采用以下Agent设计模式：

1. **Tool Use Agent**：OA Agent使用Function Calling调用7个业务工具
2. **State Machine Agent**：审批Agent基于LangGraph状态机实现4步工作流
3. **RAG Agent**：知识问答Agent基于向量检索增强LLM

## 5. 技术栈总览

| 层次 | 技术选型 |
|------|---------|
| 前端 | HTML5 + CSS3 + Vanilla JS |
| 后端框架 | FastAPI |
| LLM | DeepSeek API (deepseek-chat) |
| ORM | SQLAlchemy |
| 数据库 | SQLite（业务数据）+ ChromaDB（向量） |
| Agent框架 | LangChain + LangGraph |
| 协议 | Function Calling |
| Embedding | text-embedding-3-small |
