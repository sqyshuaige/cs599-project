# Product Spec - AgentBridge

## 1. 产品概述

### 1.1 产品定义
AgentBridge 是一个基于 Agentic AI 技术对传统企业OA系统进行智能化改造的项目。通过对现有OA办公系统的AI赋能，将传统表单操作升级为自然语言交互，实现智能审批、异常检测和知识问答功能。

### 1.2 原始系统分析（改造前）

**现有OA系统现状：**
- 员工需手动登录系统，逐表单填写请假、报销申请
- 审批流程依赖人工逐级审查，效率低下
- 公司政策分散在多个文档中，员工查找困难
- 报销审核缺乏自动化异常检测能力
- 会议室预定需人工核对时间冲突

**核心痛点：**
1. **操作繁琐**：每次申请需填写多个字段，效率低
2. **审批延迟**：人工审批受时间、精力限制，平均审批周期1-3天
3. **信息检索困难**：新员工不了解公司制度，HR需反复解答
4. **缺乏风控**：报销异常依赖人工发现，容易遗漏

### 1.3 改造目标（改造后）

| 维度 | 改造前 | 改造后 |
|------|--------|--------|
| 申请创建 | 手动表单填写 | 自然语言对话创建 |
| 审批效率 | 1-3天人工审批 | AI秒级分析 + 建议 |
| 制度查询 | 翻文档/问HR | RAG智能问答 |
| 异常检测 | 人工抽查 | AI全量分析 |
| 用户体验 | 被动操作 | 主动服务 |

## 2. 核心功能

### 2.1 AI智能对话（Function Calling）
- 自然语言查询请假/报销状态
- 对话式创建请假/报销申请
- 语音输入转文字（扩展）

### 2.2 智能审批Agent（LangGraph）
- 多步骤审批工作流：分析 → 政策审核 → 决策 → 执行
- 风险评估（低/中/高）
- 自动生成审批建议和理由
- 支持人工介入和AI自动审批

### 2.3 知识库问答（RAG）
- 向量化存储公司制度文档
- 基于语义检索的精准问答
- 支持追问和上下文理解

### 2.4 报销异常分析
- AI分析全量报销数据
- 自动标记异常报销单
- 生成风险报告和改进建议

## 3. 用户角色

- **普通员工**：创建申请、查询状态、咨询政策
- **部门经理**：审批申请、查看团队数据
- **系统管理员**：维护知识库、监控系统

## 4. 技术约束

- LLM: DeepSeek API（经济高效）
- 嵌入模型: text-embedding-3-small
- 向量数据库: ChromaDB（本地部署）
- 框架: LangGraph + LangChain
- 协议: Function Calling + MCP（扩展）
